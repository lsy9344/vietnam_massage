## 2. 1순위 작업

### WO-001. 지급완료 처리자와 변경 이력 화면 표시

연결 요구사항:

- REQ-010

현재 상태:

- 지급완료 상태, 지급시각, 처리자 ID, 변경 이력은 DB와 서버 로직에 저장됩니다.
- 일일정산 화면에는 지급완료 여부와 지급시각만 보입니다.
- 처리자와 변경 이력 목록은 화면에서 볼 수 없습니다.

작업 내용:

1. `TherapistDailySettlementPaymentHistory` 조회 경로를 추가합니다.
2. 일일정산 행의 `지급완료` 영역에 처리자 정보를 표시합니다.
3. 처리자는 최소 `paidByAccountId`를 표시하고, 가능하면 계정명 또는 직원명을 함께 표시합니다.
4. 각 마사지사 지급완료 셀에서 최근 변경 이력을 확인할 수 있게 합니다.
5. 이력에는 변경 전 상태, 변경 후 상태, 처리자, 변경시각을 표시합니다.
6. 지급완료/해제 버튼은 기존처럼 `payout:write` 권한과 잠금 월 차단을 유지합니다.

검수 기준:

- 지급완료 처리 후 화면에서 지급시각과 처리자를 확인할 수 있습니다.
- 지급완료 해제 후에도 이력에서 지급완료와 해제 이벤트를 모두 확인할 수 있습니다.
- 새로고침 후에도 지급완료 상태, 처리자, 변경 이력이 유지됩니다.
- 마감확정 또는 잠금 운영월에서는 버튼이 보이지 않거나 비활성화되고 서버에서도 변경이 차단됩니다.

권장 테스트:

- `src/modules/settlements/therapist-daily-settlement-service.test.ts`
  - payment history 조회 DTO 추가 검증
  - 지급완료, 해제 이력 순서 검증
- `tests/e2e/story-4-2-therapist-daily-settlement.spec.ts`
  - 지급완료 클릭 후 상태, 지급시각, 처리자 표시 검증
  - 완료 취소 후 이력 표시 검증

### WO-002. 콜원장 정산 정보 숨김 E2E 보강

연결 요구사항:

- REQ-004

작업 내용:

1. 카운터 또는 일반 직원 계정으로 `/calls`에 접속합니다.
2. `마사지사1수당`, `마사지사2수당`, `귀케어풀`, `콜인정` 열이 보이지 않는지 검증합니다.
3. 관리자 또는 정산 담당 계정으로 접속했을 때는 같은 열이 보이는지 검증합니다.
4. 일반 직원 화면에서 예약 등록, 결제수단 입력, 상태 변경이 깨지지 않는지 확인합니다.

검수 기준:

- 일반 직원은 정산 금액 관련 열과 상세 경고 문구를 볼 수 없습니다.
- 관리자 또는 정산 담당자는 정산 정보를 볼 수 있습니다.
- 열 숨김 때문에 그리드 입력, 저장, 자동저장이 깨지지 않습니다.

권장 테스트:

## 4. 3순위 작업

### WO-007. `예약건수` 라벨 의미 확인

연결 요구사항:

- REQ-009

현재 상태:

- 코드상 `예약건수`는 원장 전체 row 수입니다.
- 라벨은 여전히 `예약건수`라서 일반 사용자가 "예약 상태인 건수"로 오해할 수 있습니다.

확인 질문:

- 화면 라벨을 `예약건수`로 유지할지, `전체 등록건수` 또는 `원장 등록건수`로 바꿀지 의뢰자 확인이 필요합니다.

작업 방향:

- 의뢰자가 기존 문구 유지를 원하면 그대로 둡니다.
- 오해 방지를 원하면 라벨을 `원장 등록건수`로 바꾸고 설명 문구를 짧게 추가합니다.

검수 기준:

- 사용자가 방문완료, 노쇼, 취소 상태가 되어도 전체 등록 건수에서 빠지지 않는다는 의미를 이해할 수 있습니다.

### WO-008. 돈통 잔액 표시 여부 확인

연결 요구사항:

- REQ-008 확인 필요 항목

현재 상태:

- 현금, 카드, 계좌, 기타 결제수단별 합계는 구현되어 있습니다.
- `돈통 잔액` 별도 표시는 구현되어 있지 않습니다.

확인 질문:

- 돈통 잔액을 별도 KPI로 표시할지 확인해야 합니다.
- 표시한다면 계산식을 `현금 결제합계 - 현금 지출합계`로 볼지 확인해야 합니다.
- 현재 `DailyExpense`에는 결제수단 구분이 없으므로 `현금 지출합계`를 정확히 계산하려면 지출 결제수단 필드가 추가로 필요할 수 있습니다.

검수 기준:

- 의뢰자가 돈통 잔액 표시를 원하지 않으면 현재 상태로 종료합니다.
- 의뢰자가 원하면 지출 결제수단 데이터 모델 변경 여부까지 포함해 별도 작업으로 분리합니다.
