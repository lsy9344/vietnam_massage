# 베트남 에스테틱 ERP — Experience Spine

> 본 스파인과 `DESIGN.md`는 **모든 mock보다 우선한다**(충돌 시 두 스파인 WIN). 시각 아이덴티티·토큰은 `DESIGN.md`에, 행동·상태·흐름은 이 문서에 산다.

## Foundation

단일 웹앱이다 — 주인·카운터·정산이 같은 ERP를 쓰는 **데스크톱 웹**(PC 키보드 + 마우스, D1). 터치는 콜 입력 원장의 타깃이 아니다. **언어는 한국어 전용**(i18n 없음), **라이트 모드 전용**(다크 모드 없음, v1). UI 시스템은 **shadcn/ui + Tailwind CSS v4**이며, 브랜드 레이어 델타와 잠긴 status 토큰은 `DESIGN.md`가 비주얼 아이덴티티로 정의한다. ERP의 중심은 `실시간콜입력`을 대체하는 **콜 원장**이고, 모든 화면이 이 원장을 읽어 계산한다(객실 현황·TV현황판·대시보드·정산·월마감).

표면(surface)은 사이드바를 가진 일반 ERP 화면들 + **TV현황판 풀스크린 모드**(chrome/사이드바 숨김, 입력 없음, 자동 갱신, 거리 가독성용 대형 램프)다.

## Information Architecture

좌측 고정 사이드바를 도메인 그룹으로 묶는다(D4). 역할별 랜딩과 사이드바 가시성은 권한(FR-34)으로 게이트되며, 권한 없는 그룹은 비활성이 아니라 **숨긴다**.

## Voice and Tone

마이크로카피. 브랜드 보이스/미감은 `DESIGN.md`에 산다.

| Do | Don't |
|---|---|
| "종료확인 객실은 즉시 처리하세요" | "객실 상태를 확인해 주시기 바랍니다" |
| "셀 수정 즉시 자동저장" / "모든 변경 저장됨" | "변경 사항이 성공적으로 저장되었습니다 ✓" |
| "D코스는 마사지사2 필수 — 배정해야 저장됩니다" | "유효성 검사 오류가 발생했습니다" |
| "＋ 새 콜 행 추가" | "신규 레코드 생성" |
| 만근 인정 23일 · 적용 수당표 2026-06 → ₩2,000,000 | "최종지급액: 2,000,000" (맨숫자) |

규칙: 운영자가 *바로 행동하는* 짧은 한국어 라벨을 우선한다. **status 값(`사용중`·`예약`·`청소중`·`종료확인`·`빈방`)과 콜 상태(`방문완료`·`노쇼`·`취소`)는 글자 그대로 보존** — 동의어로 바꾸지 않는다. 정산·월마감은 항상 **근거**를 함께 보여주고 맨숫자를 던지지 않는다(§6.5, D6).

## Component Patterns

행동 규칙. 시각 스펙은 `DESIGN.md.Components`(또는 shadcn 기본값)에 산다.

| 컴포넌트 | 사용처 | 행동 규칙 |
|---|---|---|
| 편집 그리드 | 콜/예약 입력 원장 | `Tab`/`Shift+Tab` 셀 이동, `Enter` 아래 행 + 확정, 화살표 이동, `Esc` 편집 취소. 코드 필드(상태·코스·객실·결제수단·할인구분)는 type-ahead 드롭다운; 고객/메모/비고는 자유 텍스트. **셀 blur 시 행 단위 autosave**(저장중→저장됨, 실패 시 인라인 retry). computed 셀(결제금액·수당·귀케어풀·콜인정)은 읽기전용. 상태→`방문완료` 시 자동 계산이 행에 보이게 반영. **D코스 + 마사지사2 빈칸이면 저장/방문완료 차단 + 인라인 danger 오류**(`{colors.danger}`). `[＋ 새 콜 행 추가]` 행은 하단 고정. 그리드는 선택 날짜 단위(운영월 내), 집계는 날짜 기준이지 행 범위가 아니다. |
| 객실 상태 카드 | 첫화면 · 객실 현황 · TV현황판 | status badge는 **색상 + 텍스트 라벨 + 상태별 글리프**(●◷◐⚠○) 항상 동반(`{colors.status-*}`). `사용중`이고 남은분 0이면 표시 상태가 `종료확인`으로 전환. 종료확인 카드는 주의 상태(아래 State Patterns). 빈방은 outline 후퇴. **상태별 안내 문구(FR-18)**를 카드 필드로 노출(사용중/청소중/예약/종료확인/빈방별 웨이터 안내 — 예: 종료확인 "청소/입실 안내 대기"). 읽기 전용(첫화면/객실현황/TV 모두 입력 없음). |
| 월마감 스테퍼 | 월마감 | `작성중 → 검토중 → 마감확정 → 잠금` 수평 스테퍼, active 단계 badged. 액션 버튼은 역할 + 현재 상태로 게이트. 재오픈은 비관리자에게 보이되 비활성(visible affordance, gated action). |
| 정산 결과 + 근거 | 정산 · 월마감 미리보기 | 계산 결과와 산출 근거(만근 인정일·콜 수·적용 수당 정책·순위)를 항상 함께. 갯수왕/만근수당은 산출 근거를 화면에 표시(FR-27/FR-28). |
| 운영월/날짜 selector | 모든 데이터 화면 topbar | 운영월 + 조회날짜 선택. 변경 시 KPI·그리드·정산이 즉시 재조회(로딩 상태 동반). |

## State Patterns

| 상태 | 표면 | 처리 |
|---|---|---|
| 로딩 (콜드/조회 변경) | 대시보드 · KPI | shadcn `Skeleton`으로 레이아웃 윤곽 유지, **즉시 이해 가능한 로딩 상태**와 함께 갱신(§6.2). 화면을 멈추지 않음(서버 집계). |
| 로딩 (그리드) | 콜 원장 | 행 스켈레톤; 입력 가능 셀은 데이터 도착 후 활성. |
| 빈 상태 (빈방) | 첫화면 · 객실현황 · TV | 활성 콜 없으면 `빈방`(`{colors.status-빈방}`) + dashed 카드 + "즉시 가능" / "—". |
| 빈 상태 (데이터 없음) | 정산 · 대시보드 · 그리드 | "이 날짜의 콜이 없습니다" 등 한 문장 + 다음 행동(콜 추가/날짜 변경). |
| 오류 (autosave 실패) | 콜 원장 | 행 상태 `저장 보류`(`{colors.danger}`), 입력 유지, 인라인 retry. **절대 조용히 드롭하지 않음**(D5). |
| 오류 (D코스 검증) | 콜 원장 | 마사지사2 셀 danger ring + 행 아래 메시지 "D코스는 마사지사2 필수…". 저장/방문완료 차단(FR-14). **프로그램적 연결 필수:** 마사지사2 셀에 `aria-invalid="true"` + `aria-describedby`로 메시지 노드를 연결하고, 차단 시도 시 메시지를 `role="alert"`/`aria-live="assertive"`로 알린다 — 색상만으로 전달하지 않으며, `!` 아이콘 + 텍스트 라벨을 항상 동반(아래 Accessibility Floor). |
| 주의 (종료확인) | 첫화면 · TV현황판 | 배지=어두운 `{colors.status-종료확인}`(white 텍스트) + 카드=`{colors.status-종료확인-glow}` glow ring + **gentle pulse(소리 없음)**. 멀리서도/한눈에 읽힘. 양 표면 동일 처리(D6). **모션 안전:** pulse는 `prefers-reduced-motion: reduce`를 반드시 존중 → 애니메이션 없이 **정적 고대비 ring + ⚠ 글리프**로 폴백. pulse는 느린 opacity breathe(약 1.5~2s 주기, ~0.5Hz)이며 **초당 3회(3Hz)를 절대 넘지 않는다**(발작 안전, "blink" 아님). "⚠ 결제·확인 필요" 라벨 + 남은시간 알림 리스트에도 노출. |
| 잠금 (월마감) | 월마감 · 영향 데이터 | `잠금` 운영월은 일반 사용자의 지급 영향 데이터 수정을 막는 read-only 상태. 확정 스냅샷 기준 조회(FR-29, §6.4). 관리자 재오픈만 해제. |
| 권한 없음 | 사이드바 | 권한 없는 그룹은 숨김(비활성 아님). "차단" 화면 없음(D4). |

## Interaction Primitives

**키보드 우선 그리드 입력이 제품의 핵심이다** — 엑셀 입력 속도를 잃으면 실패다(SM-C2).

- `Tab` / `Shift+Tab` — 다음/이전 셀
- `Enter` — 아래 행 이동 + 확정(commit)
- 화살표 — 셀 이동 · `Esc` — 편집 취소
- 코드 셀은 type-ahead 드롭다운(첫 글자 입력 시 필터)
- **autosave commit-on-blur** — 셀 blur 시 행 단위 저장 + 재계산, 저장 버튼 없음
- **방문완료 전환 시 자동 계산** — 상태→`방문완료`면 결제금액·할인·마사지사1/2 수당·귀케어 풀·콜인정이 행에서 즉시 계산되어 read-only 셀에 표시(FR-12). 그 외 상태는 0/—.
- **월마감 확정 = 이중확인** — 확정 전 지급 총액 + 대상 인원/구성 요약 + "확정 시 스냅샷이 고정되어 이후 설정 변경으로 재계산되지 않습니다" 경고를 보여주고, 두 번째 확인을 받는다(D6, FR-29). **포커스 관리(shadcn Dialog, 비정상 popover 금지):** 모달 오픈 시 포커스를 다이얼로그 내부의 **안전한(비파괴) 컨트롤**(제목 또는 [취소])로 이동, 포커스를 모달 안에 trap, `Esc`로 1단계 취소, 닫힐 때 포커스를 트리거([마감 확정])로 복귀. 파괴적 확정 버튼에 초기 포커스를 두지 않는다. `role="alertdialog"` + 경고 라벨링.
- **재오픈 게이트** — 관리자 권한 + 사유 입력 필수. 비관리자에게는 보이되 비활성(FR-30).
- **종료확인 pulse 모션 안전** — `prefers-reduced-motion: reduce` 시 애니메이션을 끄고 정적 고대비 ring + ⚠ 글리프로 폴백. pulse 주기 ≤0.5Hz, 절대 3Hz 초과 금지(발작 안전).

**금지:** 카운터 그리드에 무거운 애니메이션/화려한 시각화(SM-C2), 색상만으로의 상태 표현, 모달 2단 중첩, 콜 입력 화면 터치 최적화(데스크톱은 키보드 타깃).

## Accessibility Floor

행동 기준. 시각 대비는 `DESIGN.md`(shadcn AA 기본값 상속, 브랜드 오버라이드 검증).

- **상태는 절대 색상(hue)만으로 전달하지 않는다** — 배지는 항상 **색상 + 텍스트 라벨 + 상태별 글리프** 세 가지를 동반한다. 청소중 gold·빈방 bronze·종료확인 orange는 색맹(적/녹색맹)에서 유사한 따뜻한 색 띠로 수렴하므로, **hue만으로 구분하지 않도록 상태별 글리프를 부여**한다: 사용중 ●(filled) · 예약 ◷(clock) · 청소중 ◐(turnover) · 종료확인 ⚠(alert) · 빈방 ○(empty). badge·칩·룸 카드·TV현황판·그리드 드롭다운 스와치 모두 라벨+글리프 필수(스와치 색 점이 단독으로 의미를 나르지 않음).
- **폼 오류는 색상만으로 전달하지 않는다** — D코스 인라인 검증은 마사지사2 셀에 `aria-invalid="true"` + `aria-describedby`로 메시지를 연결하고 `role="alert"`로 알린다. danger ring 외에 `!` 아이콘 + 텍스트 메시지를 항상 동반.
- 그리드는 완전 키보드 조작 가능(`Tab`/`Enter`/화살표/`Esc`), `Tab` 순서 = 읽기 순서.
- **월마감 이중확인 모달**은 포커스를 trap하고, 초기 포커스를 안전한(비파괴) 컨트롤에 두며, `Esc`로 취소, 닫힐 때 트리거로 포커스 복귀(shadcn Dialog `role="alertdialog"`).
- **TV 모드 거리 가독성** — 대형 램프(`{typography.tv-room-name}` 등)로 멀리서 객실명·상태·남은시간·코스·담당 판독(§5.4). 거리에서도 글리프가 hue 보조.
- 포커스 링은 brand 색 inset ring으로 명확히 표시(`{colors.brand}`), 라이트 캔버스 대비 AA 유지.
- 종료확인 pulse는 시각 전용·소리 없음이며 `prefers-reduced-motion: reduce` 시 정적 ring으로 폴백하고 3Hz를 넘지 않는다(발작 안전). 텍스트 라벨("⚠ 결제·확인 필요")로 비시각 사용자도 인지 가능.

## Responsive & Platform

v1은 **두 표면(surface)만** 타깃한다. 모바일/태블릿 네이티브 타깃은 없다.

- **데스크톱 웹 (1차 표면)** — 주인·카운터·정산 모두 PC 키보드+마우스로 쓰는 조밀한 ERP(D1). 최소 폭 가정 **≥1280px**(콜 원장 12열 그리드 + 사이드바 228px가 들어가는 밀도). 사이드바 + topbar + 조밀 데이터 레이아웃. 터치 최적화는 입력 원장 대상 아님.
- **TV현황판 풀스크린 (2차 표면)** — 같은 웹앱의 fullscreen 모드(D2). chrome/사이드바/입력 없음, **자동 갱신**(콜 원장에서 흘러온 상태를 폴/푸시로 주기 갱신, 마지막 갱신 시각·갱신 지연 표시). DESIGN.md의 대형 타이포 램프(`{typography.tv-room-name}` 40px / `{typography.tv-status}` 28px / `{typography.tv-meta}` 22px)로 **10~15ft 거리 가독성**을 만든다. 대비는 다크 캔버스가 아니라 큰 글자 + 채도 status 블록 + 글리프로 만든다.
- **없음(v1):** 네이티브 모바일/태블릿, 다크 모드, i18n. 단일 따뜻한 라이트 팔레트·한국어 전용.

## Key Flows

### Flow 1 — 주인 noah, 실시간 현황으로 매장 파악 (UJ-1, 오후 매장 피크)

1. noah가 로그인하면 **첫화면 실시간 현황**이 랜딩(주인 = full 사이드바, 읽기 권한).
2. 11개 객실 카드가 상태 색 우선으로 한눈에 스캔된다: 사용중(emerald)·예약(blue)·청소중(gold)·빈방(bronze).
3. 하단 KPI(오늘 예약 38 / 방문완료 31 / 노쇼 3 / 결제합계 ₩4,120,000)와 코스별 콜, 남은시간 알림을 본다.
4. **클라이맥스:** 202·401 호실 카드가 `{colors.status-종료확인}` orange로 빛나며 gentle pulse — "⚠ 결제·확인 필요". noah는 별도 엑셀 시트를 열지 않고, 어느 방이 지금 손이 필요한지 *한 화면에서* 즉시 판단한다. 카운터에게 한마디 던지고 다시 운영으로 돌아간다.

실패: 데이터 갱신 지연 → 카드에 스켈레톤 + "갱신 중", 직전 값 유지. 조용히 빈 화면 금지.

### Flow 2 — 카운터 mina, 콜 입력 → 방문완료 (UJ-2, 손님 퇴실 시점)

1. mina는 로그인 후 **콜/예약 입력 원장**에 랜딩. 오늘 날짜 그리드 + 일별 요약 strip.
2. 401 호실 D코스 행을 입력하다 마사지사1만 넣고 상태를 바꾸려 한다 → 마사지사2 셀에 danger ring + "D코스는 마사지사2 필수" → 저장 보류. mina가 마사지사2를 type-ahead로 배정.
3. 손님이 퇴실하자 상태 셀을 `방문완료`로 바꾼다(드롭다운, focus ring).
4. **클라이맥스:** `Enter` 순간 결제금액(할인 −₩50,000 반영 ₩100,000)·마사지사1/2 수당·귀케어 풀·콜인정 1이 read-only 틴트 셀에 *엑셀 수식처럼* 즉시 채워지고, 행 상태가 "저장중…→저장됨"으로 바뀐다. 같은 콜 원장을 읽는 첫화면·객실현황·정산이 곧바로 이 값을 반영한다 — 누가 무엇을 바꿨는지는 감사 로그에 남는다.

실패: autosave 실패 → 행 "저장 보류" + 입력 유지 + 인라인 retry. 다시 `Enter`로 재시도.

### Flow 3 — 정산 담당 seora, 월마감 확정 (UJ-4, 월말)

1. seora가 **월마감**으로 들어가 운영월 2026-06을 연다. 스테퍼: `작성중 → 검토중 → 마감확정 → 잠금`, 현재 `검토중`.
2. **미리보기**에서 마사지사별 월 총콜·정산액·만근 인정일·만근수당·갯수왕 순위/수당·최종지급액, 운영팀 월인센, 귀케어 합계를 본다 — 각 수치 옆에 *산출 근거*(만근 23일 / 적용 수당표 2026-06 / 40콜 이상 1위)가 붙어 있다.
3. 검토 후 [마감 확정]을 누른다.
4. **클라이맥스(이중확인):** 모달이 지급 총액 + 대상 인원/구성 요약과 함께 "**확정 시 스냅샷이 고정되어 이후 설정 변경으로 재계산되지 않습니다**" 경고를 띄운다. seora가 두 번째 확인을 누르는 순간 마감 스냅샷이 저장되고 운영월이 `잠금`으로 전환 — 이후 수당표·직원명·원장을 바꿔도 이 지급액은 흔들리지 않는다. 확정/잠금은 감사 로그에 기록된다.

실패/되돌리기: 잘못 확정 → 재오픈은 **관리자 + 사유 입력**으로만 가능(seora가 정산 담당이면 관리자에게 요청). 비관리자에게 재오픈 버튼은 보이되 비활성.

### Flow 4 — 웨이터 도윤, 객실 상태/안내 문구 확인 (UJ-3, 객실 전환 순간)

1. 도윤이 로그인하면 **객실 현황**에 랜딩한다 — 운영 현황의 **읽기 전용 subset**(사이드바는 조회 항목만, 콜 원장·정산·설정 없음). 원장 편집 권한 없음(FR-34).
2. 11개 객실 카드를 **상태 색상 + 라벨 + 글리프**로 스캔한다: 사용중(emerald ●)·예약(blue ◷)·청소중(gold ◐)·빈방(bronze ○). 각 카드에 **남은시간**과 상태별 **안내 문구(FR-18)**가 붙어 있다(FR-16/FR-17).
3. 305 호실이 `사용중`에서 남은분 0이 되며 **`종료확인`(⚠)으로 전환** — 카드가 어두운 orange 배지 + glow로 떠오르고 안내 문구가 "청소/입실 안내 대기"로 바뀐다.
4. **클라이맥스:** 도윤은 원장을 만지지 않고 **상태 + 남은시간 + 종료확인 + 안내 문구**만 읽어, 305는 곧 청소·입실 안내가 필요하고 옆 빈방은 즉시 안내 가능함을 한눈에 판단한다 — 홀에서 손님을 어느 방으로 안내할지 즉시 결정한다.
5. 안내 후 도윤은 화면을 그대로 두고(자동 갱신), 다음 전환을 기다린다.

실패: 상태 갱신 지연 → 카드에 "갱신 지연" + 마지막 갱신 시각 표시, 직전 값 유지. 멈춘 보드를 빈 화면/무표시로 두지 않는다(웨이터가 오래된 상태로 잘못 안내하는 것 방지). 도윤은 편집 권한이 없으므로 상태를 직접 바꿀 수 없고, 갱신은 카운터의 콜 원장 입력에서 흘러온다.

## 권한·감사 (Permissions & Audit) — 제품 특화 섹션

엑셀 전환의 핵심 개선점은 **변경 추적**과 **마감 후 흔들림 방지**다(FR-34/FR-35, §6.4). 행동 규칙:

- **권한 게이트(FR-34):** 화면 접근과 기능 실행 범위를 역할로 분리. 사이드바 가시성은 권한 기반(숨김), 지급/수당/마감/직원 영향 기능은 권한 검사 통과 필수.
- **감사 로그(FR-35):** 콜 상태·결제/할인·담당자·출퇴근·수당표·직원·월마감 확정/취소/재오픈 변경은 행위자·액션·대상·변경 전/후·시각을 불변 기록. 일반 운영 경로에서 삭제 불가. 감사 로그는 도메인 계산을 대신하지 않고 *이력만* 보존.
- **월마감 스냅샷 불변성(§6.4 / SM-4):** 확정 스냅샷은 이후 설정 변경으로 자동 재계산되지 않는다. 월간 KPI도 확정 후에는 스냅샷 기준으로 조회. 재오픈만이 잠금을 해제하며, 관리자 + 사유를 요구하고 감사 로그에 남는다.
