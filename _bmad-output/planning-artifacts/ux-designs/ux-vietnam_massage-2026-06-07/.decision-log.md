## Decisions

### D1 — Foundation: form-factor, input, tone, language (2026-06-07)
- **Surfaces:** 데스크톱 웹 (주인·카운터·정산) — single web app. User framed the whole thing as a "웹앱". Did NOT explicitly pick the large-TV or tablet surfaces from the menu → TV현황판 delivery needs clarification (PRD FR-19 still requires it). [see D2]
- **Counter input device:** PC 키보드 + 마우스 → design for **high density + keyboard-first** input; preserve Excel-speed entry (SM-C2). Touch is not a target for the input ledger.
- **Tone (main ERP):** **Light** (밝은 배경).
- **Language:** **한국어 전용** (Korean only). No i18n layer for v1.

### D2 — TV현황판 delivery (2026-06-07)
- **Delivery:** Same web app, **fullscreen "TV mode"** screen (not a separate app). Shares the exact room-state calculation and color tokens with 객실현황 (FR-16~19).
- **Tone:** **Light**, same as main ERP. → entire product is single-tone light; no dark mode for v1.
- **Implication:** TV mode needs a large-typography / high-contrast *layout* variant on a light background — contrast carried by size + saturated status color blocks, not by a dark canvas.

### D3 — Brand direction + status color system (2026-06-07)
- **Brand direction:** "따뜻한 스파" — warm spa, gold/amber/earth luxury on a warm light canvas. Owner-delight goal: "선명하고 고급스러운".
- **Chosen theme:** **② 로얄 골드 (Royal Gold)** — rich & saturated gold, the most vivid "money" read. (Artifact: `.working/color-themes-1.html`, variation ②. Full hex set documented there; distilled into DESIGN.md at Finalize. Spine wins on conflict.)
- **Core status color tokens (semantic, locked across ALL surfaces incl. TV mode):**
  - `사용중` → emerald green `#178A5A` (active / occupied — the only green)
  - `예약` → royal blue `#2F6FD0` (reserved — the only blue)
  - `청소중` → bright gold `#D9A526` (turnover — gold family)
  - `종료확인` → electric orange `#F25C1F` (**attention / action-needed — the only orange, must pop**)
  - `빈방` → dim bronze `#B3A37D` (idle — desaturated neutral, recessive)
  - Brand primary: deep gold `#B57E12` on rich cream background.
- **Rule:** these status tokens are identical in 첫화면 카드, 객실현황, TV현황판, and dashboard chips. Call states (방문완료/노쇼/취소) and danger (D코스 검증 오류) hex carried in the artifact → DESIGN.md.
- **No dark mode** (v1). Single warm-light palette everywhere.

### D4 — Information architecture & navigation (2026-06-07)
- **Nav model:** **Left fixed sidebar**, grouped by domain. Persistent, collapsible-optional.
- **Domain groups (sidebar order):** 운영 현황(첫화면 실시간 현황 · 객실 현황 · TV 현황판) → 콜 원장(콜/예약 입력 · 일별 지출/요약) → 정산(마사지사 · 귀케어 · 운영팀 인센) → 월마감 → 대시보드(오늘 KPI · 월간 KPI · 주인용 그래프) → 마스터 설정(운영월·객실·직원·코스·수당/인센·코드·시간슬롯) → 감사 로그.
- **TV 현황판** opens as a fullscreen mode (chrome/sidebar hidden) from within 운영 현황.
- **Role-based landing** (FR-34 permissions gate both landing + sidebar visibility):
  - 주인/관리자 → 첫화면 실시간 현황 (full sidebar)
  - 카운터 → 콜/예약 입력 원장
  - 정산 담당 → 정산 화면
  - 웨이터 / 조회 전용 → 객실 현황 (read-only subset of sidebar)
- Sidebar items render per role permission; unauthorized groups hidden, not just disabled.

### D5 — Counter call-input ledger: editable grid (2026-06-07)
- **Pattern:** **Editable spreadsheet grid** (rows = calls, columns = 콜 fields). Excel muscle-memory preserved (SM-C2 / §6.5).
- **Behavioral spec (→ EXPERIENCE.md):**
  - Keyboard-first: `Tab`/`Shift+Tab` between cells, `Enter` to next row, arrow keys to move; type-ahead in dropdown cells (상태/코스/객실/결제수단/할인구분).
  - Inline dropdowns for coded fields; free text for 고객/메모/비고.
  - `[+ 새 콜 행 추가]` row pinned at the bottom of the day's grid.
  - When 상태 → `방문완료`: 결제금액·마사지사1/2 수당·귀케어 풀·콜인정 auto-compute in the row, visibly (read-only computed cells distinct from input cells).
  - **D코스 inline validation (FR-14):** if 코스=D and 마사지사2 empty, block save/방문완료 with an inline cell error using the danger token; do not silently save.
  - Grid scoped to a selected 날짜 (within current 운영월); month aggregation by date, never by row range.
- **Save:** **Autosave per cell** — commit + recompute on cell blur, no save button. Per-row save status shown subtly (saving / saved / error) so the Excel feel holds; failed autosave surfaces an inline retry, never silently drops input.

### D6 — Key interactions & state patterns (2026-06-07)
- **종료확인 alert (첫화면 + TV현황판):** orange status token **+ gentle pulse/blink** on the card. Visual-only, no sound — readable at a glance and from a distance without being noisy. Same treatment on both surfaces.
- **월마감 확정 (FR-29):** **two-step confirm with a summary** — before locking, show 지급 총액 + 대상 인원/구성 summary and an explicit "확정 시 스냅샷이 고정되어 이후 설정 변경으로 재계산되지 않습니다" warning, then a second confirm. 재오픈 (FR-30) requires 관리자 + 사유 입력, and is shown but disabled for non-admins (visible affordance, gated action).
- **월마감 state presentation:** horizontal stepper reflecting `작성중 → 검토중 → 마감확정 → 잠금` with the active state badged; action buttons gated by role + current state.
- **Settlement screens (§6.5):** always show 계산 결과 **and** 산출 근거 together (e.g. 만근 인정일 수, 콜 수, 적용 수당 정책) — never a bare number.
