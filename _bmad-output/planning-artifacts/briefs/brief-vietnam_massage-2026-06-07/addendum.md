## 반드시 보존할 핵심 업무 흐름

### 예약/콜 입력

- 하루 100개 슬롯, 월 최대 31일 구조의 예약/콜 입력 기능을 보존한다.
- 상태값은 `예약`, `사용중`, `청소중`, `방문완료`, `노쇼`, `취소`를 지원한다.
- 입력 필드는 시간, 방번호, 코스, 고객/메모, 마사지사1, 마사지사2, 귀케어담당, 예약상태, 할인구분, 결제수단, 비고, 확인값을 포함한다.
- 방문완료 상태일 때만 결제금액, 마사지사 수당, 귀케어 풀, 콜인정이 발생한다.
- 일별 지출금액, 내용, 담당자, 비고 입력 기능을 보존한다.

### 객실 현황과 TV 표시

- 11개 객실의 현재 상태, 코스, 담당, 시작시간, 남은분, 종료예정, 안내 문구, 메모를 표시한다.
- 객실별 최신 `사용중`, `청소중`, `예약` 건을 기준으로 상태를 계산한다.
- `사용중`이고 남은 시간이 0이면 `종료확인`으로 표시한다.
- TV현황판은 입력 없는 전체화면 조회 전용 카드 화면으로 구현한다.
- 상태별 색상 표현은 `사용중`, `청소중`, `예약`, `종료확인`, `빈방` 기준으로 보존한다.

### 정산

- 마사지사가 `마사지사1` 또는 `마사지사2` 어느 칸에 배정되어도 담당 콜로 인정한다.
- 마사지사 당일정산은 개인별 코스 수당표를 기준으로 계산한다.
- 출근/퇴근 시간이 자정을 넘으면 퇴근시간에 24시간을 더해 대기시간을 계산한다.
- 대기시간 8시간 이상이면 만근 인정으로 처리한다.
- B코스 귀케어 풀은 해당일 정상 근무 귀케어사에게 균등 분배한다.
- 운영팀 일일 인센은 총콜 30/40/50콜 기준으로 지급하며 정상 근무자만 대상이다.
- 운영팀 월 인센은 월 총콜 1,000~1,500콜 구간별 금액을 팀장 30%, 카운터팀 35%, 웨이터팀 35%로 분배한다.

### 월마감

- 마사지사별 월 총콜, 월 정산액, 만근 인정일, 만근수당, 갯수왕 순위/수당, 최종지급액을 산출한다.
- 만근수당은 만근 인정일 20일 이상이면 2,000,000으로 처리한다.
- 갯수왕은 월 총콜 40콜 이상 대상자 중 1~3위에 각각 5,000,000 / 3,000,000 / 1,000,000을 지급한다.
- 운영팀 월 총콜, 월인센 전체, 일일인센 합계, 귀케어 지급 합계, 마사지사 최종 지급 합계를 함께 보여준다.
- 마감 결과는 확정 시점의 스냅샷으로 저장하고 잠가야 한다.

## 이관 리스크와 보존 기준

- 일부 월간 요약 수식은 실제 31일차 입력 슬롯 전체와 범위가 맞지 않는다. ERP 집계는 반드시 날짜 조건으로 구현한다.
- 시간 목록이 두 곳에 중복되어 있으며 한쪽은 01:00까지, 다른 쪽은 02:00까지 포함한다. 단일 시간 슬롯 마스터가 필요하다.
- 직원명, 객실명, 코스명이 문자열로 참조된다. ERP에서는 고유 ID를 만들고 표시명 변경 이력을 보존한다.
- 수당/가격/인센 기준은 과거 월마감 결과에 영향을 주지 않도록 적용월과 스냅샷을 가져야 한다.
- D코스는 2인 코스이나 엑셀은 마사지사2 입력을 강제하지 않는다. ERP에서는 정책 확인 후 필수 검증을 적용한다.
- 할인구분은 값이 있으면 고정 100,000 할인으로 계산된다. 금액 다양화는 별도 정책 확정 전까지 1차 범위에 넣지 않는다.
- 주요 변경 작업은 감사 로그에 남긴다: 상태 변경, 결제/할인 변경, 담당자 변경, 출퇴근 변경, 수당표 변경, 직원 변경, 월마감 확정/취소/재오픈.

## 개발 착수 전 확인 질문

| 항목 | 확인 질문 |
| --- | --- |
| 시간 슬롯 | 예약 가능 시간이 01:00까지인지, 02:00까지인지 |
| 할인 정책 | 모든 할인구분을 100,000 고정 할인으로 유지할지 |

| 마사지사 수당 | 마사지사5~50의 0원 수당이 실제 값인지 초기값인지 |
| 월마감 재오픈 | 재오픈 권한자와 절차를 어떻게 둘지 |
| 객실 표시명 | `1번방`과 `101 호실` 중 표준 표시명을 무엇으로 할지 |
