# Product Brief: 베트남 에스테틱 ERP

## 요약

이 제품은 기존 `sheet.xlsx`로 운영하던 베트남 에스테틱/마사지 매장의 업무를 웹 기반 ERP로 이전하는 프로젝트다. 새 아이디어를 검증하는 제품이 아니라, 이미 운영 중인 엑셀 업무 시스템을 기능 누락 없이 보존하고 데이터베이스, 권한, 감사 로그, 월마감 잠금이 가능한 운영 시스템으로 재구축하는 것이 목적이다.

원본 엑셀은 단순 예약표가 아니다. `실시간콜입력`을 중심 원장으로 두고 예약/콜 입력, 객실 상태, TV 현황판, 오늘 대시보드, 운영팀 인센, 귀케어 정산, 마사지사 정산, 월마감, 직원/객실/코스/수당 설정까지 포함한다. ERP 1차 버전의 성공은 새로운 기능의 양이 아니라, 기존 엑셀의 화면·입력·계산·마감 기능이 삭제되거나 약화되지 않았음을 증명하는 데 있다.

## 해결할 문제

현재 운영은 하나의 복잡한 엑셀 파일에 의존한다. 12개 시트, 월 31일 x 일 100개 슬롯, 숨김 행/열, 조건부서식, 이름 정의, 수만 개 수식이 업무 규칙을 담고 있어 변경과 검증이 어렵다. 담당자명·객실명 같은 문자열이 사실상 키로 사용되고, 마감 후에도 현재 수식과 설정 변경에 따라 지급액이 다시 흔들릴 수 있으며, 누가 언제 어떤 값을 바꿨는지 추적하기 어렵다.

ERP 전환의 가장 큰 위험은 “새 시스템을 만들면서 기존 기능 일부가 빠지는 것”이다. 특히 정산 계산, 객실 상태 표시, 숨김 보조 목록, 일별 지출, 월마감 보너스처럼 엑셀 안에 분산된 기능은 화면상 작아 보여도 실제 운영에서는 필수 기능이다.

## 제품 방향

ERP는 엑셀 셀 좌표를 그대로 복제하지 않고, 엑셀에 들어 있는 업무 규칙을 정규화된 데이터와 계산 엔진으로 옮긴다. 중심은 `service_calls` 예약/콜 원장이며, 직원·객실·코스·수당·인센 기준은 독립 마스터로 관리한다. 방문완료 상태만 매출, 콜 인정, 정산에 반영하고, 객실 현황과 TV 표시판은 원장의 최신 상태를 조회해 실시간으로 보여준다.

원본의 기능은 보존하되, 엑셀의 취약점은 ERP 전환 시 개선한다. 월간 집계는 행 번호 범위가 아니라 날짜 조건으로 계산하고, 직원/객실/코스는 이름이 아닌 고유 ID로 참조하며, 수당과 인센 기준은 적용월과 스냅샷을 가져야 한다. 월마감은 미리보기, 검토, 확정, 잠금 단계로 처리하고 확정 후 변경은 재오픈 권한과 감사 로그를 통해서만 가능해야 한다.

## 주요 사용자

- 관리자: 설정, 직원, 수당, 인센, 월마감, 재오픈을 관리한다.
- 카운터: 예약/콜 입력, 상태 변경, 결제수단과 할인 정보를 입력한다.
- 웨이터: 객실 상태, 남은 시간, 안내 문구를 확인한다.
- 정산 담당: 마사지사, 귀케어, 운영팀 정산과 월마감을 검토한다.
- 조회 전용 사용자: 오늘 대시보드와 TV/객실 현황을 확인한다.

## 1차 범위

1차 ERP에는 기존 엑셀의 12개 시트가 담당하던 기능이 모두 포함되어야 한다.

- 마스터 설정: 기준월, 직원, 객실, 코스, TV 표시명, 시간 슬롯, 상태/결제/할인 코드, 마사지사별 코스 수당, 운영팀 인센 기준
- 예약/콜 원장: 일자별 예약/방문/사용중/청소중/방문완료/노쇼/취소 입력, 담당자 배정, 할인, 결제수단, 확인값, 비고
- 자동 계산: 방문완료 기준 결제금액, 할인금액, 콜인정, 마사지사1/2 수당, 귀케어 풀, 일별 지출과 순매출
- 운영 화면: 오늘 대시보드, 웨이터 객실 현황, TV 현황판, 상태별 색상/안내 문구, 남은 시간과 종료확인 표시
- 정산 화면: 운영팀 근무/일일·월 인센, 귀케어 일일정산, 마사지사 일일정산, 출퇴근/대기시간/만근 인정
- 월마감: 마사지사 월 총콜, 월 정산액, 만근수당, 갯수왕 수당, 운영팀/귀케어 합계, 최종지급액, 확정/잠금
- 운영 안전장치: 권한, 감사 로그, 설정 변경 이력, 월마감 스냅샷, 데이터 이관 검증

## 명시적 비범위

1차 구축은 기존 엑셀 기능 보존이 우선이다. 신규 CRM, 마케팅 자동화, 회계 시스템 연동, 모바일 앱, 복잡한 할인 정책 확장, 고객 멤버십 기능은 별도 요구가 확정되기 전까지 1차 범위에 넣지 않는다. 단, 향후 확장을 막지 않도록 데이터 모델은 고유 ID, 이력, 적용월 구조를 가져야 한다.

## 성공 기준

- 원본 `sheet.xlsx`의 12개 시트와 숨김 시트 `목록`까지 모두 ERP 기능 또는 설정으로 매핑된다.
- `실시간콜입력`의 A:S 입력/계산 구조와 일별 지출 구조가 ERP에서 재현된다.
- 방문완료 기준 매출, 할인, 콜인정, 마사지사 수당, 귀케어 풀 계산이 원본과 일치한다.
- 웨이터리스트와 TV현황판의 객실 상태, 남은 시간, 종료확인, 상태별 색상 표현이 재현된다.
- 운영팀 인센, 귀케어 N분의1 정산, 마사지사 일일정산, 만근수당, 갯수왕 수당, 월마감 최종지급액이 원본 규칙과 일치한다.
- 월마감 확정 후에는 설정이나 원장 변경이 과거 지급액을 임의로 바꾸지 않는다.
- 기능 보존 추적표와 샘플 계산 대조 테스트를 통해 “누락 기능 0개”를 검증한다.

## 확인 필요 사항

- 예약 가능 시간은 `01:00`까지인지, `02:00`까지인지 확정해야 한다.
- 할인구분이 있으면 모두 `100,000` 고정 할인으로 유지할지 확인해야 한다.
- D코스에서 마사지사2 입력을 ERP에서 필수로 강제할지 확인해야 한다.
- 정상 근무 귀케어사가 0명인 날의 귀케어 풀 처리 방식을 정해야 한다.
- 마사지사5~50의 코스 수당 0원이 실제 운영값인지 초기 입력값인지 확인해야 한다.
- 월마감 확정 후 재오픈 권한과 절차를 정해야 한다.
- `1번방` 형식과 `101 호실` 형식 중 ERP 표준 객실 표시명을 확정해야 한다.
