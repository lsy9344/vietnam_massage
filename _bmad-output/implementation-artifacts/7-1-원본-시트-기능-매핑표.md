# Story 7.1: 원본 시트 기능 매핑표

Status: done

<!-- Note: Validation completed during create-story. -->

## Story

As a 운영자 또는 QA 담당자,
I want 원본 엑셀의 모든 시트가 ERP 기능으로 어떻게 이전됐는지 확인하기를,
so that v1에서 기능 누락 없이 엑셀 업무가 보존됐는지 검증할 수 있다.

## Acceptance Criteria

1. 시트 기능 매핑표는 원본 `오늘대시보드`, `실시간콜입력`, `웨이터리스트`, `TV현황판`, `운영팀근무인센`, `귀케어일정산`, `마사지사일정산`, `월마감`, `직원DB`, `TV설정`, `설정_코스수당`, 숨김 시트 `목록`을 모두 포함한다. 누락된 시트가 있으면 검증 실패로 표시한다.
2. 각 시트 행은 ERP 화면, 설정, 계산 엔진, 검증 항목 중 하나 이상에 연결된다. "이관됨" 같은 모호한 설명만으로 통과 처리하지 않는다.
3. 숨김 시트 `목록`과 드롭다운 원천은 상태, 결제수단, 할인구분, 확인값, 운영팀/귀케어 근무상태, 시간 슬롯, 객실/방번호, 코스, 마사지사, 귀케어 담당 매핑을 확인할 수 있어야 한다.
4. `실시간콜입력` 행은 콜 원장 입력, 방문완료 계산, 일별 지출, 일별 요약, 코스별 요약과 연결된다. A:S 입력/계산 구조와 U:X 지출/요약 구조를 구분해 설명한다.
5. `웨이터리스트`와 `TV현황판` 행은 객실 상태 계산, 남은 시간, 종료확인, 상태 색상/표시값, TV fullscreen 화면과 연결된다. 두 화면이 `src/modules/rooms/room-status-service.ts`의 같은 상태 계산을 사용한다는 점을 표시한다.
6. 정산 관련 시트 `운영팀근무인센`, `귀케어일정산`, `마사지사일정산`, `월마감`은 일정산, 운영팀 인센, 월마감 스냅샷, 계산 대조 항목과 연결된다. 모든 정산/매출/콜인정 설명은 `방문완료` 기준 계산 규칙을 명시한다.
7. 마스터 관련 시트 `직원DB`, `TV설정`, `설정_코스수당`, `목록`은 직원, 객실, 코스, 수당/인센 정책, 코드/시간 슬롯 마스터와 연결된다. 표시명과 고유 ID 분리 원칙을 표시한다.
8. 개발자 또는 QA가 매핑 검증을 실행하면 숨김 `목록`을 포함한 12개 전체 시트 포함 여부, ERP 기능 연결, 코드값 반영, 계산 대조 항목 연결을 검사하고 모두 통과해야 한다.

### Architecture Compliance

- Keep business/calculation logic in existing domain modules. Story 7.1 should reference existing calculations and mappings; it must not duplicate payment, settlement, room-status, monthly-close, or dashboard calculations.
- The mapping source may be static TypeScript data because the workbook inventory is project evidence. Do not add a database table or Prisma migration unless a later story requires editable migration tracking.
- Use Server Components by default. A Client Component is only needed for local expand/collapse/filter interactions.
- Route data should be deterministic and testable without a database connection. The mapping table should render even when local DB is unavailable.
- No Server Action, audit write, mutation UI, monthly close state change, seed reset, or payout-impacting write belongs in this story.
- If adding an Epic 7 module folder, keep it narrowly scoped to migration/verification evidence. Do not put shared business calculations there.
- Use existing visual language: dense ERP table, small summary counters, text labels with any status color. Do not create a marketing page or hero.
- Preserve RBAC: administrator can access; optional QA/read-only access must be exact-route and read-only.
