# Story 4.6: 운영팀 월 인센 미리보기

Status: done

<!-- Note: Validation completed during create-story. -->

## Story

As a 정산 담당,
I want 운영월 총콜 기준 운영팀 월 인센을 미리 계산해 보기를,
so that 월마감 전에 운영팀 지급 예상액과 분배 근거를 검토할 수 있다.

## Acceptance Criteria

1. 정산 담당자가 운영팀 월 인센 미리보기 화면에서 운영월을 선택하면 해당 운영월의 월 총콜, 적용 threshold, 전체 월인센, 팀별 분배, 직원별 분배가 표시된다.
2. 월 총콜은 선택 운영월 `startDate`부터 `endDate`까지의 calls domain 완료 콜 계산 결과 중 `calculationStatus === "calculated"`인 `방문완료`/`VISIT_COMPLETE` row의 `opsCallCredit` 합계다. `예약`, `사용중`, `청소중`, `노쇼`, `취소`, `course_policy_missing`, `therapist_rate_missing`, `second_therapist_required` row는 월 총콜에 포함하지 않고 warning/제외 근거로만 표시한다.
3. 적용월에 활성 `OpsMonthlyIncentiveRule`이 있으면 월 총콜이 충족한 가장 높은 `thresholdCallCount`를 적용한다. seed 기준 구간은 1000/1100/1200/1300/1400/1500콜이며, 전체 월인센 금액은 정책의 `totalAmount`다.
4. 적용월에 `OpsMonthlyIncentiveRule`이 없으면 조용히 0으로 가정하지 않는다. 결과 DTO는 `ruleStatus: "missing_policy"`와 한국어 warning을 반환하고 화면에는 정책 없음/미확정 근거를 표시한다.
5. 정책은 있지만 월 총콜이 최저 threshold 미만이면 `ruleStatus: "below_threshold"`, 전체 월인센 0 VND, 구간 미달 근거를 표시한다.
6. 전체 월인센이 계산되면 정책의 분배율을 적용해 팀장 30%, 카운터팀 35%, 웨이터팀 35% 금액을 산출한다. seed 기준과 다른 정책 값이 DB에 있으면 hard-code가 아니라 정책 row의 `leadShare`, `counterTeamShare`, `waiterTeamShare`를 사용하고 화면에 분배율을 함께 표시한다.
7. 팀장 몫은 active `Employee.employeeGroup === "OPERATIONS"` 중 position/seed 기준 팀장 대상에게 배정한다. 카운터팀과 웨이터팀 몫은 각 active 운영팀 직원 내부에서 균등 분배하고, 나누어떨어지지 않는 VND 잔여는 deterministic order(`sortOrder`, `staffCode`, `Employee.id`)로 1 VND씩 배분한다.
8. 직원 식별과 분배 row key는 `Employee.id`다. 표시명, 직책 텍스트, Excel 행 번호, staffCode만을 downstream 저장 키로 쓰지 않는다.
9. 이 story는 월 인센 미리보기 결과를 저장하지 않는다. `OpsMonthlyIncentivePreview`/payout result table, 직접 수정 UI/action, 감사 로그, `/closing` 월마감 snapshot 생성/수정, 운영월 상태 변경은 만들지 않는다.
10. 운영월이 `작성중` 또는 `검토중`이면 현재 콜 원장과 현재 정책 기준의 미확정 미리보기임을 표시한다. 운영월이 `마감확정` 또는 `잠금`이면 확정 스냅샷과 혼동하지 않도록 "현재 기준 미리보기"와 "확정값은 월마감 스냅샷 기준" 안내를 분리해 표시하며, 스냅샷은 자동 재계산하지 않는다.
11. `/settlements/operations/monthly`는 `requireRouteAccess("/settlements/operations/monthly")` 또는 동일 `/settlements` prefix 경계로 보호된다. 화면은 조회 전용이므로 write action을 만들지 않지만, 권한 없는 counter/waiter/read-only 사용자는 기존 route access 정책대로 차단/리다이렉트된다.
12. 월 총콜 방문완료 기준, 1000~1500 threshold, 구간 미달, 정책 없음, 30/35/35 분배, 팀 내부 잔여 배분, 직원 stable ID, 미확정/확정 안내, no persistence/no closing side effect를 검증하는 unit/static/e2e 테스트가 통과한다.
