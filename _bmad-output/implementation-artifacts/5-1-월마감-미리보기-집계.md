# Story 5.1: 월마감 미리보기 집계

Status: done

<!-- Note: Validation completed during create-story. -->

## Story

As a 정산 담당,
I want 운영월의 마사지사, 운영팀, 귀케어 지급액을 미리보기로 확인하기를,
so that 월마감 확정 전에 지급 대상과 산출 근거를 검토할 수 있다.

## Acceptance Criteria

1. 정산 담당자가 `/closing` 월마감 화면에 접근해 운영월을 선택하면 해당 운영월의 월마감 미리보기가 표시되고, 모든 집계는 `OperatingMonth.startDate`부터 `endDate`까지의 운영월 날짜 범위 기준으로 계산된다.
2. 마사지사 preview는 활성 마사지사별 `Employee.id`, `staffCode`, `displayName`, 월 총 담당 콜, 월 정산액, 만근 인정일 기초 상태, 최종지급액 기초값을 표시한다. Story 5.1의 최종지급액 기초값은 월 정산액과 같고, 만근수당/갯수왕 수당은 Story 5.2 전까지 `미적용` 또는 `후속 story 대기` 근거로 분리 표시한다.
3. 마사지사 월 총콜과 월 정산액은 기존 `listTherapistDailySettlements()` 일별 결과를 운영월 날짜별로 집계해 만든다. UI component가 콜 수, 코스별 수당, 담당자 역할, 수당 정책 상태를 재계산하지 않는다.
4. 운영팀 preview는 기존 `listOpsDailyIncentives()` 일별 결과를 운영월 날짜별로 집계해 일일인센 합계를 만들고, 기존 `listOpsMonthlyIncentivePreview()`로 월 총콜, 적용 threshold, 월인센 전체, 팀/직원별 월인센 산출 근거를 가져온다.
5. 귀케어 preview는 기존 `listEarcareDailySettlements()` 일별 결과를 운영월 날짜별로 집계해 귀케어 지급 합계, 정상근무자 수, 지급 대상자 수, 정상근무자 0명/미분배 같은 warning 근거를 표시한다.
6. 월마감 preview 요약은 마사지사 지급 합계, 운영팀 일일인센 합계, 운영팀 월인센 합계, 귀케어 지급 합계, 전체 지급 합계를 표시하고, 하위 row 합계와 요약 합계가 일치해야 한다.
7. 모든 직원 row key와 downstream 식별자는 `Employee.id`다. 표시명, 직책, `staffCode`, Excel 행 번호는 화면/근거 표시용으로만 사용한다.
8. 화면은 계산 결과와 산출 근거를 함께 표시한다. 최소 근거는 기간, source day count, included/excluded call count, policy/warning counts, 대표 evidence를 포함하며 맨숫자 금액만 단독으로 표시하지 않는다.
9. 운영월 상태가 `작성중` 또는 `검토중`이면 "미확정 미리보기"로 표시한다. `마감확정` 또는 `잠금`이면 이 화면은 "현재 기준 미리보기"이며 확정 스냅샷 값이 아니라는 안내를 표시한다.
10. Story 5.1은 읽기 전용이다. `MonthlyClose`, snapshot, payout persistence table, 운영월 상태 변경, 월마감 확정/잠금/재오픈 action, 감사 로그 생성, 지급액 직접 수정 UI를 만들지 않는다.
11. `/closing` page는 `requireRouteAccess("/closing")`로 보호된다. administrator와 settlement_manager만 접근 가능해야 하며, counter/waiter/read_only_viewer는 기존 route access 정책대로 redirect된다.
12. 월마감 미리보기 로딩 중에는 route segment loading UI 또는 Skeleton을 표시해 운영월 변경 시 이전 운영월 값과 새 운영월 값을 혼동하지 않게 한다. 조회 실패 시 `role="alert"`와 재조회 affordance를 제공한다.
13. unit/static/e2e 테스트는 운영월 날짜 범위 집계, 마사지사 월 총콜/정산액, 운영팀 일일/월 인센, 귀케어 합계, 전체 지급 합계 일치, 산출 근거/warning 표시, 미확정/현재 기준 표시, read-only/no snapshot boundary, 권한 redirect를 검증한다.

### Architecture Compliance
