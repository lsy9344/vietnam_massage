# Story 5.6: 월마감 이중확인 모달과 접근성

Status: done

<!-- Note: Validation completed during create-story. -->

## Story

As a 정산 담당,
I want 월마감 확정 전에 지급 총액과 스냅샷 불변 경고를 이중확인하기를,
so that 실수로 확정해 과거 지급액을 잠그는 위험을 줄일 수 있다.

## Acceptance Criteria

1. 운영월 상태가 `검토중`이고 월마감 미리보기가 준비되어 있을 때 `/closing`의 `마감 확정` 클릭은 즉시 `confirmMonthlyCloseAction`을 제출하지 않고 이중확인 모달을 연다.
2. 모달에는 운영월, 지급 총액, 대상 인원/구성 요약, 주요 지급 항목(마사지사, 운영팀 일일/월 인센, 귀케어)이 표시된다. 값은 `listMonthlyClosingPreview()` 결과를 재사용하고 UI에서 월마감 계산을 재구현하지 않는다.
3. 모달에는 `확정 시 스냅샷이 고정되어 이후 설정 변경으로 재계산되지 않습니다` 또는 동등한 짧은 한국어 불변성 경고가 표시된다.
4. 모달에서 `취소`, close affordance, 또는 `Esc`를 누르면 모달이 닫히고 월마감 확정은 실행되지 않는다. 운영월 상태, `MonthlyClosing`, 감사 로그는 변경되지 않는다.
5. 모달의 두 번째 확인 버튼을 누를 때만 기존 Story 5.3 `confirmMonthlyCloseAction`이 실행된다. 성공 후 `/closing`은 revalidate되어 운영월 상태 `마감확정`, 최신 `확정 스냅샷`, `잠금` action을 표시한다.
6. 모달은 shadcn/ui `AlertDialog` 또는 기존 shadcn Dialog 기반으로 구현하되 alert dialog semantics를 제공해야 한다. `role="alertdialog"`가 노출되고 제목/설명이 screen reader에 연결된다.
7. 모달이 열릴 때 초기 포커스는 제목 또는 `취소` 같은 안전한 컨트롤에 놓인다. destructive 확정 버튼에 초기 포커스를 두지 않는다.
8. 모달이 열려 있는 동안 키보드 포커스는 모달 내부에 trap된다. 모달이 닫히면 포커스는 `마감 확정` 트리거로 돌아간다.
9. 확정 요청이 서버 권한, 상태 전이, 스냅샷 생성, audit rollback 등으로 실패하면 모달 또는 월마감 action 영역에 안전한 한국어 오류 메시지가 `role="alert"`/동등한 live semantics로 표시되고, 사용자는 확정이 완료되지 않았음을 알 수 있다.
10. unit/static/e2e 테스트는 모달 열림, 즉시 제출 방지, 지급 요약/불변성 경고, Esc 취소 무부작용, safe initial focus, focus trap, focus return, alertdialog labeling, 확정 성공, 확정 실패 표시, 기존 Story 5.3/5.5 confirm/reopen 흐름 호환을 검증한다.

### Architecture Compliance

- `closing` domain service가 확정 상태 전이, snapshot creation/versioning, audit payload를 계속 소유한다.
- Server Action은 authenticate/authorize/validate/domain call/revalidate/error mapping만 수행한다.
- Client component는 modal state, preview DTO display, form submission state만 소유한다.
- 월마감 금액/인원 summary는 preview DTO를 표시하는 read-only UI다. UI에서 지급액 계산을 다시 만들지 않는다.
- `closing:write` 권한은 기존대로 confirm 권한이며, `closing:reopen`과 섞지 않는다.
