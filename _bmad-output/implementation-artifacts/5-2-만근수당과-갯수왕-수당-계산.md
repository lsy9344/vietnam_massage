# Story 5.2: 만근수당과 갯수왕 수당 계산

Status: done

<!-- Note: Validation completed during create-story. -->

## Story

As a 정산 담당,
I want 월마감에서 마사지사 만근수당과 갯수왕 수당을 계산하기를,
so that 월 최종지급액에 보너스성 지급액을 원본 기준대로 반영할 수 있다.

## Acceptance Criteria

1. `/closing` 월마감 미리보기의 마사지사 row는 `Employee.id` 기준으로 월 정산액, 만근 인정일 수, 만근수당, 갯수왕 순위, 갯수왕 수당, 최종지급액을 표시한다.
2. 만근 인정일 수는 Story 4.1의 마사지사 일별 출퇴근/대기시간 결과 중 `대기시간 >= 8시간`으로 인정된 일자만 운영월 날짜 범위 안에서 집계한다. 현재 repo에는 Story 4.1 산출물과 `TherapistAttendance` persistence가 없으므로, 구현 전 해당 source가 없으면 임의 계산/더미 값을 만들지 않고 Story 4.1 선행 필요를 명시적으로 드러낸다.
3. 만근 인정일이 19일 이하인 마사지사는 `fullAttendanceAllowanceAmount = 0`이고, 화면 근거에는 `20일 미만` 또는 동등한 한국어 제외 사유가 표시된다.
4. 만근 인정일이 20일 이상인 마사지사는 `fullAttendanceAllowanceAmount = 2_000_000`이고, 화면 근거에는 인정일 수와 `20일 이상 2,000,000 VND` 기준이 표시된다.
5. 갯수왕 대상자는 Story 5.1에서 집계한 마사지사 `totalCallCount` 기준 40콜 이상인 마사지사만 포함한다. 40콜 미만 row는 순위/수당이 없고 제외 근거가 표시된다.
6. 갯수왕 순위는 월 총콜 내림차순으로 1~3위만 부여하고, 동률 tie-breaker는 deterministic 해야 한다. 기본 tie-breaker는 `totalCallCount desc`, `monthlySettlementAmount desc`, `staffCode asc`, `Employee.id asc`다.
7. 갯수왕 수당은 1위 5,000,000 VND, 2위 3,000,000 VND, 3위 1,000,000 VND이다. 대상자가 3명보다 적으면 존재하는 대상자에게만 수당을 적용하고 없는 순위 row를 생성하지 않는다.
8. 마사지사 최종지급액은 `monthlySettlementAmount + fullAttendanceAllowanceAmount + countKingBonusAmount`이며, 마사지사 지급 합계와 전체 지급 합계는 개별 row 합계와 일치한다.
9. 화면은 보너스 금액만 단독으로 보여주지 않고 만근 인정일, 20일 기준, 월 총콜, 40콜 기준, 순위, tie-breaker 근거를 함께 표시한다.
10. Story 5.2는 현재 기준 미리보기 계산만 확장한다. `MonthlyClose`, snapshot/payout persistence table, 운영월 상태 변경, 확정/잠금/재오픈 Server Action, 감사 로그 생성은 만들지 않는다.
11. `/closing` route guard는 `requireRouteAccess("/closing")`를 유지하며 administrator와 settlement_manager만 접근할 수 있다. UI에 보너스가 표시되어도 counter/waiter/read_only_viewer 권한은 넓히지 않는다.
12. unit/static/e2e 테스트는 만근 19일/20일, 40콜 미만/이상, 1~3위 수당, 대상자 3명 미만, 동률 tie-breaker, 최종지급액/요약 합계 반영, Story 4.1 source 부재 처리, no snapshot/no audit/no write boundary를 검증한다.

### Architecture Compliance

- `src/app`은 route composition과 display only다. 보너스 계산, ranking, final payout totals는 `src/modules/closing` domain service가 소유한다.
- `settlements`/Story 4.1 source가 만근 판정을 소유한다. `closing`은 month-level count와 allowance application만 소유한다.
- `Employee.id`가 모든 downstream merge key다. `displayName`, `staffCode`, `position`, Excel 행 번호는 표시/근거 전용이다.
- 날짜는 ISO `YYYY-MM-DD`, 금액은 formatted string이 아닌 integer VND number로 DTO에 담는다.
- Story 5.2는 read-only preview다. Prisma `create`, `update`, `delete`, `$transaction`, `recordAuditEvent`, Server Action write path, `MonthlyClose`/snapshot model은 scope 밖이다.
- `shared`에 새 helper를 추가하지 않는다. count-king ranking과 allowance constants는 현재 `closing` preview에서만 쓰이므로 service-local이 맞다.
