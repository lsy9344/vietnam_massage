# Story 6.4: 대시보드 로딩/실패/차트 색상 규칙

Status: done

<!-- Note: Validation completed during create-story. -->

## Story

As a 주인/관리자,
I want 대시보드가 로딩, 실패, 빈 상태와 차트 색상 의미를 일관되게 처리하기를,
so that 시각화가 운영 판단을 돕고 콜 입력이나 상태 의미를 방해하지 않게 할 수 있다.

## Acceptance Criteria

1. `/dashboard/today`, `/dashboard/monthly`, `/dashboard/reports`는 데이터 로딩 중 `src/components/ui/skeleton.tsx`의 shadcn Skeleton 또는 동등한 layout-preserving loading UI를 표시한다. 화면은 blank screen처럼 보이면 안 되고 `aria-busy`와 한국어 로딩 label을 유지한다.
2. 사용자가 조회날짜 또는 운영월을 변경하면 canonical URL search params와 서버 조회 기준이 일치해야 한다. 새 기준 로딩 중에는 이전 기준값을 새 기준값처럼 보이게 두지 않고, loading skeleton/기준 label로 전환 기준을 구분한다.
3. `/dashboard/*/error.tsx` error boundary는 raw `error.message`/stack/digest를 사용자에게 노출하지 않는다. 안전한 한국어 오류 문구, `role="alert"`, `다시 시도` reset affordance, `현재 조건 새로고침` router refresh affordance를 제공한다.
4. 데이터가 없는 조회 기준은 명확한 empty state로 표시한다. 날짜/운영월 no-call, calculated 완료 콜 없음, 정산 source 없음, room status 없음, snapshot missing은 가능한 한 서로 다른 메시지로 구분하고, 존재하지 않는 값을 임의의 0값 완료 그래프로 꾸미지 않는다.
5. `사용중`, `예약`, `청소중`, `종료확인`, `빈방` status color/token은 해당 객실 상태 의미를 표시할 때만 사용한다. 매출, 순위, 지급 구성, 노쇼/취소 등 일반 chart series는 status token을 임의 재사용하지 않는다.
6. chart legend, tooltip, SVG title, bar/line label, table/list fallback은 색상만이 아니라 텍스트 라벨과 숫자를 함께 제공한다. `StatusBadge`는 색상, 글리프, 텍스트 라벨을 함께 유지한다.
7. 대시보드 차트는 기존 Story 6.3 project-local SVG/CSS primitive를 우선 유지한다. chart library를 새로 도입하지 않는다. 도입이 불가피하면 구현 시점에 registry/docs로 exact version을 확인하고, React 19/Next 16 호환과 bundle impact를 validator에 명시한다.
8. 대시보드 상태 처리 검증은 loading, 기준 변경, 실패, empty state, snapshot missing, status color 의미 보존, legend/tooltip/text label, table fallback, route access, 성능 회귀를 포함한다.

### Architecture Compliance

- Dashboard routes는 Server Component 중심과 URL search params selection state를 유지한다. Redux/Zustand/TanStack Query/React Query를 새로 도입하지 않는다.
- Loading은 App Router route segment `loading.tsx`와 existing `Skeleton` primitive를 사용한다. 별도 loading library를 추가하지 않는다.
- Error boundary는 App Router `error.tsx` Client Component convention을 유지한다. 사용자는 안전한 한국어 오류만 보고 raw exception detail은 보지 않는다.
- Chart component는 presentation-only scaling과 percentages만 계산할 수 있다. 콜 row, snapshot JSON, 정산 evidence, 객실 active call selection을 직접 계산하지 않는다.
- Status color tokens are locked: `사용중`, `예약`, `청소중`, `종료확인`, `빈방` 의미를 표시할 때만 사용한다. Revenue/ranking/payout/no-show/cancel colors must not use `bg-status-*` or `var(--color-status-*)`.
- No mutation UI, Server Action, audit write, monthly close status change, DB migration, or payout-impacting write belongs in this story.
- `shared` helper는 실제로 today/monthly/reports가 모두 재사용하고 테스트 비용을 줄일 때만 추가한다. 단일 route chart helper는 route-local로 둔다.
