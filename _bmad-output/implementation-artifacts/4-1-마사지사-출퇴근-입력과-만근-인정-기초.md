# Story 4.1: 마사지사 출퇴근 입력과 만근 인정 기초

Status: done

<!-- Note: Validation completed during create-story. -->

## Story

As a 정산 담당,
I want 마사지사별 일일 출퇴근 시간을 입력하고 대기시간과 만근 인정 여부를 계산하기를,
so that 마사지사 일일정산과 월마감 만근수당의 기초 데이터를 정확히 만들 수 있다.

## Acceptance Criteria

1. 정산 담당자가 `/settlements` 마사지사 일정산 화면에서 운영월과 날짜를 선택하면 활성 마사지사 50명의 일별 출퇴근 입력 목록이 표시되고, 직원 식별은 마사지사 이름이 아니라 `Employee.id` 기준으로 처리된다.
2. 정산 담당자가 마사지사 출근시간과 퇴근시간을 저장하면 출근시간, 퇴근시간, 근무일자, 운영월, 마사지사 ID가 저장되고, 시간은 표시 문자열이 아니라 계산 가능한 minute-of-day 값으로 보존된다.
3. 퇴근시간이 출근시간보다 빠르면 자정 이후 퇴근으로 계산하며, 자정 넘김 때문에 대기시간이 음수로 계산되지 않는다.
4. 대기시간이 8시간 이상이면 해당 일자는 만근 인정으로 표시되고, 8시간 미만이면 만근 인정으로 표시되지 않는다.
5. 출퇴근 입력값이 누락되었거나 형식이 잘못되면 저장은 차단되거나 미완료 상태로 표시되고, 안전한 한국어 오류 메시지가 표시된다.
6. 운영월이 `마감확정` 또는 `잠금` 상태이면 서버에서 출퇴근 시간 수정이 차단되고 잠긴 운영월 안내가 표시된다.
7. 출퇴근 시간이 생성, 수정, 삭제 또는 비활성 처리되면 행위자, 대상 마사지사, 변경 전후값, 날짜, 시각이 감사 로그로 기록되고 지급액 영향 변경으로 분류된다.
8. 월마감 미리보기의 만근수당 계산은 Story 4.1의 `TherapistAttendance` source와 공개 service를 재사용하며, 콜 수, 귀케어 근무상태, 운영팀 근무상태, 더미 값으로 만근 인정일을 추정하지 않는다.
9. 50명 목록, 출퇴근 저장, 자정 넘김, 8시간 이상/미만 만근 인정, 입력 오류, 잠금 차단, 감사 로그, 월마감 만근수당 재사용 테스트가 통과한다.

### Architecture Compliance

- FR20-FR25 map to `src/app/(erp)/settlements` and `src/modules/settlements`; calculation logic belongs in domain services, not React components. [Source: `_bmad-output/planning-artifacts/architecture.md` Requirements to Structure Mapping]
- Use Next.js App Router Server Actions for internal UI mutations and keep actions thin: authenticate/authorize, validate, call domain service, map `ActionResult<T>`. [Source: `_bmad-output/planning-artifacts/architecture.md` API & Communication Patterns]
