작성일: 2026-06-10

## 요약

## 완료 결과

### Story 7.1: 원본 시트 기능 매핑표

`src/modules/migration/sheet-feature-mapping.ts`가 숨김 `목록`을 포함한 원본 12개 sheet coverage의 source of truth가 됐다. `/masters/sheet-mapping`은 read-only QA/reference route로 시작했고, `read_only_viewer`는 exact route만 접근하도록 제한했다.

### Story 7.2: 핵심 계산 대조 검증 fixture와 테스트

`tests/fixtures/migration-calculation-comparison.ts`와 read-only memory Prisma adapter가 계산 대조의 기준이 됐다. 방문완료 결제/할인/콜인정, D코스 2인 검증, 객실/TV 상태, 운영팀/귀케어/마사지사 정산, 월마감 최종지급액은 기존 domain service 경계를 재사용해 검증했다.

### Story 7.3: 이관 검증 리포트와 누락 추적

`src/modules/migration/migration-verification-report.ts`가 Story 7.1 mapping과 Story 7.2 mismatch shape를 결합하는 report DTO를 제공한다. `MigrationVerificationIssue`와 `MigrationVerificationIssueHistory`가 persisted QA tracking을 소유하며, route-local Server Action은 `migration:write` 권한으로만 상태 변경을 허용한다.

## 잘 된 점

- 원본 12개 sheet와 숨김 `목록`의 coverage가 코드 source of truth, 화면, 문서, validator로 연결됐다.
- Excel 셀 좌표나 workbook range를 런타임 구현 기준으로 쓰지 않는 원칙이 Story 7.1~7.3 전반에 유지됐다.
- 계산 대조는 migration module에 새 계산 엔진을 만들지 않고 calls, rooms, settlements, closing domain service를 재사용했다.
- Story 7.2의 mismatch report shape가 Story 7.3 report DTO로 이어져 fixture, 테스트, 리포트 간 계약이 일관됐다.
- QA tracking persistence를 operational audit log와 분리했다. 감사 로그는 운영 변경 이력이고, migration verification history는 출시 전 검증 상태 이력이라는 경계가 명확해졌다.
- DB/browser가 없어도 실행 가능한 source guardrail과 unit coverage를 유지했고, DB-backed browser tests는 별도 조건으로 분리했다.

## 어려웠던 점

- Epic 7은 “문서성 기능”처럼 보이지만 실제로는 source mapping, 계산 fixture, report DTO, 권한, Prisma tracking, validator가 모두 맞물리는 cross-cutting epic이었다.
- Story 7.2의 핵심 계산 대조와 Story 7.3의 리포트 표시 사이에 “실시간 테스트 실행”과 “fixture 계약 요약”을 구분해야 했다. 현재 리포트는 요청 시 테스트를 실행하지 않고 fixture/mismatch 계약과 persisted issue를 결합한다.
- `read_only_viewer` 접근을 열면서도 `/masters` prefix 전체를 열지 않는 exact route 보안 경계를 계속 확인해야 했다.
- Full browser E2E는 DB/dev server 전제가 필요했다. Epic 7은 이를 core acceptance의 필수 경로로 만들지 않고 DB-free 검증과 source guardrail로 핵심 위험을 관리했다.
- `npx tsc --noEmit`은 이전부터 남아 있던 call-ledger keyboard, account-service enum, older E2E typing 문제 때문에 계속 전체 통과 상태가 아니다. Epic 7 변경의 type fallout은 story review에서 별도로 수습됐다.

## 핵심 학습

1. Migration verification은 문서가 아니라 product surface다. 운영자와 QA가 `/masters/sheet-mapping`에서 기능 보존율, 계산 대조, 열린 위험을 확인할 수 있어야 한다.
2. Excel evidence와 ERP implementation key를 분리한 결정은 맞았다. 셀 좌표와 range는 출처 문자열로 보존하고, 실행 로직은 stable ID와 domain service에 의존해야 drift가 줄어든다.
3. Report DTO는 계산 엔진이 아니다. 리포트는 Story 7.1 mapping, Story 7.2 fixture/mismatch contract, persisted QA issue를 결합하는 read model로 유지해야 한다.
4. QA tracking은 audit log와 다르다. 출시 전 누락/불일치 추적은 별도 model과 bounded status가 있어야 운영 감사 이력과 섞이지 않는다.
5. Source guardrail은 sandbox 제약을 보완하지만 대체재의 범위를 명확히 해야 한다. DB-backed browser coverage는 별도 실행 조건을 문서화하는 편이 더 정직하다.

## Significant Discoveries

- Epic 7의 migration verification은 단순 문서 산출물이 아니라 권한, route, DTO, persistence, validator가 있는 product capability다.
- QA issue tracking은 audit log로 대체하면 안 된다. 운영 감사와 출시 전 검증 상태는 목적과 보존 의미가 다르다.
- 현재 `/masters/sheet-mapping` report는 Story 7.2 fixture/mismatch contract를 표시한다. 요청 시 계산 테스트를 실행하거나 test result artifact를 ingest하는 구조는 아니다.
- 숨김 `목록`은 visible sheet와 같은 가중치의 100% gate로 처리하는 구현이 맞다.

## Readiness Assessment

- Testing and quality: Story 7.1~7.3 validators, focused unit tests, DB-free source guardrails, `npm run lint`, `npm run test:unit`, Prisma validation 기록이 있다. Full browser E2E는 DB/dev server 전제가 필요하며 일부는 skip 또는 환경 blocker로 기록됐다.
- Deployment: repository 내 production 배포 완료 증거는 확인하지 않았다.
- Stakeholder acceptance: 별도 명시 문서는 확인하지 않았다.
- Technical health: migration module boundary, exact read-only route access, admin-only `migration:write`, no runtime Excel parsing, QA tracking persistence는 코드와 문서가 대체로 일치한다.
- Unresolved blockers: full TypeScript check pre-existing failures, DB-backed browser execution 환경, Story 4.1 backlog가 남아 있다.

## Commitments

- Migration verification은 `src/modules/migration`과 `/masters/sheet-mapping`을 기준 경계로 유지한다.
- 숨김 `목록`은 계속 100% gate로 취급한다.
- Excel workbook range와 cell coordinate는 evidence 문자열로만 보존하고 런타임 parse를 추가하지 않는다.
- QA tracking은 audit log가 아니라 `MigrationVerificationIssue`/`MigrationVerificationIssueHistory`가 소유한다.
- DB-backed browser validation과 DB-free source guardrail은 계속 분리해서 기록한다.
